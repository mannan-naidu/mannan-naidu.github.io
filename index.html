03 / Work

01

Published — ICTEAH 2026, Taylor & Francis

Tuberculosis Detection Beyond Dataset Bias

Existing TB detectors quietly overfit to their training dataset's visual distribution and break on scans from a different hospital. The question was whether you could specifically engineer that gap away.

I am the primary author and built all four competing architectures for the comparison: the baseline CNN, Dual-Stream Fusion (ResNet50 + autoencoder with SMOTE class balancing), Context-Aware Diffusion, and the ROI-guided diffusion model — the core contribution. The ROI system directs diffusion augmentation specifically at diagnostically important lung regions rather than applying it globally, which is what drives robustness under dataset shift. Cross-dataset evaluation on an independent 4,200-image set (never seen during training): 98.45% accuracy · 94.46% recall · 95.32% F1 — best across all four architectures.

Stack: PyTorch · ResNet50 · autoencoders · SMOTE · diffusion augmentation · Vision Transformers

02

Final-year major project — live demos

MARS-Fin — regime-specific financial forecasting

Most forecasting papers report a single average accuracy and call it done. That number hides failures during market stress — exactly when a forecast matters most. We wanted a rigorous answer to which architecture holds up when volatility regimes shift.

Compared LSTM and Lag Llama (a transformer-based foundational time series model) on two deliberately contrasting NSE stocks: Adani Enterprises for event-driven shocks (the 2023 Hindenburg impact specifically) and Asian Paints for slower commodity-linked cycles. A GARCH/EGARCH model partitions Jan 2015–Dec 2024 price history into distinct volatility regimes; both architectures are evaluated separately within each regime on a multi-modal dataset — OHLCV, news sentiment, quarterly fundamentals, macro indicators.

Open forecast demo → Volatility dashboard → GitHub →

Stack: LSTM · Lag Llama · GARCH/EGARCH · XGBoost · SHAP · Streamlit · Python

Personal project — forecasting research

Innergy — energy consumption forecasting with TSF

I wanted hands-on experience with Time Series Foundation models — pretrained transformer architectures for time series that you fine-tune rather than build from scratch, similar to how LLMs changed NLP. Energy data is a natural fit: strong seasonality, clear weather correlation, real-world utility.

Built on two datasets: the IIITD institutional energy dataset (Springer Nature) and a 26,000-row energy-weather correlation dataset spanning 2018–2023. Models learn the relationship between consumption patterns and weather variables — temperature, humidity, wind. Dataset not committed to the repo due to size.

07

Personal project — backend architecture

Pathify — travel recommendation engine

Built specifically to learn Redis and Celery properly — not just to list them on a resume but to understand when a cache layer and async workers actually change what's possible. Travel recommendations are a good stress-test: lots of repeated queries that benefit from caching, and recommendation jobs that shouldn't block the request thread.

Redis on Upstash for session/caching, Celery workers for async recommendation processing, backend on Render, frontend deployed separately. The cache serves repeat queries instantly; the worker queue handles heavier jobs without blocking the user.
